# Suggestions for Project Maintenance and Guides

Based on the recent build for `test_5` and previous tests, here are actionable suggestions to improve the guides and the project structure:

## 1. Centralize the Web Application (Avoid Duplication)
**Issue**: Currently, `index.html`, `style.css`, `app.js`, and `run_quiz.bat` are being copy-pasted into every new `test_N` folder. This means if we want to add a new feature (like a timer or CSV export) or fix a bug in `app.js`, we have to update it across `test_1` through `test_5`.
**Suggestion**: 
- Create a shared `src/` or `web/` folder in the root directory holding the master `index.html`, `style.css`, and `app.js`.
- Refactor the tests so they only contain the `questions.json` and `images/` directory.
- `index.html` could take a URL parameter (e.g., `?test=test_5`) to dynamically load the appropriate `questions.json` and images, meaning we only ever run one local server for the entire project.

## 2. Updates to `llm_csv_extraction_guide.md`
**Encoding (BOM) Warnings**: Add a specific instruction recommending `encoding='utf-8-sig'` instead of just `utf-8` when reading the CSV with Python. Hebrew CSVs generated by Windows tools often include a Byte Order Mark (BOM). Standard `utf-8` reads the BOM as a character, which can break column header matching or regex operations on the first column.
**Index Mapping**: Explicitly state that when merging the CSV answers with `questions_raw.json`, the CSV provides 1-based correct answers (e.g., option 2), but `questions.json` requires a 0-based `correctIndex` (e.g., index 1). A small Python script snippet demonstrating this merge would be highly beneficial.

## 3. Updates to `hebrew_pdf_conversion_guide.md`
**Port Conflict Strategy**: The guide currently recommends manually hardcoding a unique port in `run_quiz.bat` per test (e.g., 8001, 8002). To prevent humans/agents from losing track and accidentally colliding, suggest a deterministic mapping rule: e.g., `Port = 8000 + Test Number`. For `test_5`, use `8005`.
**Vision LLM Nuances**: In Part 4 (Vision LLM path), emphasize that when LLMs fail to perfectly transcribe complex graphs with Hebrew axes, the agent should fall back to the `Cropper.js` manual UI workflow (which was excellently documented in Step E). Ensure agents know they can transcribe the text manually using their native vision capabilities if the automated Python API fails or hallucinates.

## 4. Run Scripts
Change the `run_quiz.bat` to a centralized script (like `run_all.bat`) that can launch a master menu UI to let the user select which test they want to take, rather than having to navigate into individual folders to start disparate web servers.
